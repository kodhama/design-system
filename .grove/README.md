# .grove/ — design-system's grove dials

grove-managed (regenerated by `/grove:refresh`). This directory holds the
consumer-authoritative dials grove agents read against this repo, plus the
grove-owned gate machinery. It explains the **effect** of each dial so a repo
reader needs no plugin to understand the knobs — the normative operating model
itself is never restated here (`adr-0008`; `adr-0026` D7).

- **`gates.toml`** — yours: who is *required* at each of the four gates
  (`intent` / `spec` / `build` / `ship`), `human` or `agent`. The floor
  (`adr-0018`): at least one human intent-locus gate (`intent = human` OR
  `ship = human`), validated on every run-sequencing read. Switch presets with
  `/grove:set-profile`, or hand-edit a single row. This repo's profile is
  `steward` (intent + ship human; spec + build agent).
- **`config.toml`** — yours: the shared role tokens (`adr-0026` D3) grove
  agents resolve at use time — test/typecheck/lint commands, corpus dirs,
  rubric paths, PR-contract and parked-item conventions. Values are **verified
  priors**: an agent verifies before relying, discloses a mismatch loudly, and
  routes the fix back here rather than silently working around a stale value.
  An explicit "none — …" is an honest value, not a gap. (design-system has no
  package.json or test runner — verification is grep-based over `tokens.css` /
  `patterns.md` / `icons/`; the config records that honestly.)
- **`agents/<role>.md`** — yours, optional: per-role addenda (local rules,
  worked examples) a generic `grove:<role>` agent reads when present and
  replaces with judgment when absent. Seed nothing empty.
- **`internal/`** — grove's: the gate-profile floor-guard machinery
  (`internal/gates/`, `adr-0018` D8) and the C1 enforcement defaults
  (`internal/enforcement.toml`, `adr-0018` D4), re-copied verbatim on refresh.
  Not a consumer surface — do not hand-edit.

**The fleet is plugin-carried.** The thirteen agent roles are **not** vendored
into this repo. They load automatically as `grove:<role>` subagents from the
installed grove plugin (`adr-0026` D1) — nothing lives in `.claude/agents/` for
them. The version this repo ratifies is stamped `grove plugin@<version>` in the
repo's `CLAUDE.md` managed block; a role-start mismatch against the installed
plugin is disclosed, never enforced (`adr-0026` D4).

The operating model (lifecycle enum, versioning grammar, edge taxonomy) ships
in the grove plugin — per the grove lifecycle / versioning / relations
companions, `plugin@<stamp>` (the stamp in this repo's `CLAUDE.md`). No copy of
the companions is installed here (`adr-0026` D7).
